# Spec v2 — T2-N: negotiation-response tier

Supersedes the T3 multi-turn draft. Rewritten around the adversarial review
(t3_spec_review_sol.md, verdict RETHINK): every BLOCKER is addressed by a
design change below; the review is the rationale document, this is the
decision document. Renamed T2-N because SPEC.md's T3 contract (multi-document,
cross_doc_override) is unchanged and still owed separately — this tier is a
negotiation AXIS on single documents, and public reports must never conflate
the two. [Review F16]

## Premise that survives review

The counterparty is never simulated at runtime. The response is forward-
generated by the mutation engine as a canonical PATCH LEDGER in the same
operation that logs ground truth.

## Episode shape: two phases, one episode [F11]

Phase 1 — the agent reviews the seeded document exactly as T2 today and
finalizes positions; the env returns opaque `position_id`s. Phase 2 — only
after phase-1 finalize, the env reveals the pre-generated counterparty
response and the agent dispositions it. Cross-turn continuity is scored:
`review_change` requires the relevant `prior_position_id`, which appears
NOWHERE in the phase-2 view — an agent that skipped phase 1 or ignores its
own positions cannot link. At least one counter decision per task depends on
a deterministic context slot disclosed only in phase 1. A `turn2_only`
ablation gate (≤60% of honest) enforces that the state-tracking claim is
measured, not asserted.

## The patch ledger (build FIRST, with its validators) [F12, F13]

Typed operations (`replace`, `insert_before`, `insert_after`, `delete`) over
stable section/paragraph IDs; every child patch records change_id, parent
event_id, doc/section, precondition hash, before/after text, offsets in the
canonical phase-1 document. The tracked-changes markdown view is DERIVED from
the ledger; V14 replays the ordered patches against the phase-1 document and
requires byte equality with the shipped phase-2 document, plus exact 1:1
between rendered blocks and child patches and n:1 child→parent event mapping.
Schemas + V14 + the review_change action/card schema ship before any
generator work (SPEC invariant: validators before generator).

## Event taxonomy and ground truth

Each event carries an emitted `expected_disposition` per child patch:

- concession — benign; expected accept
- regression — harmful; expected reject (ONE review_change, not reject+re-flag) [F8]
- sneaky_reinsert — parent with two children: accept the compliant original-
  location patch AND reject the inserted patch; parent credit requires both [F8, F12]
- counter_proposal — emitted ONLY on rules carrying attorney-signed
  `acceptable_counter_templates` / `unacceptable_counter_templates`; the
  template's signed label becomes `expected_decision` at emit time; an
  emit-time separability gate (max-M2-similarity vs alternatives ≥0.75 for
  acceptable, 0.00 for unacceptable, ambiguous band rejected) validates
  fixtures, never scores agents [F4, F6, F7]
- new_deviation — emitted as a tracked change, dispositioned via
  review_change like everything else [F8]
- benign_canary_change — a compliant edit generated from a signed template
  under a canary rule; expected accept; rejecting/flagging it under the
  canary rule_id ⇒ GAMED. V9 extended over all turn-event artifacts: canary
  rules own no harmful event, ever [F15]

## Action contract [F8, F9, F10]

Everything with a change_id is dispositioned ONLY via
`review_change {change_id, decision, prior_position_id, rule_id?+exact_quote?
(required for reject), proposed_redline?}`. `flag_issue` is reserved for
violations in UNCHANGED text (separate issue-ID namespace). Episode state
keeps `reviews_by_change_id`: repeated identical submissions collapse to one;
conflicting submissions are a conformance failure and a wrong disposition.
The T2-N card schema REQUIRES `changes` (additionalProperties: false per
entry) and must equal the interactive map exactly. `review_change` counts as
substantive for bounce/conformance. Scoring consumes the canonical event map,
never a pooled quote-keyed list.

## Scoring [F1, F2, F3, F5]

Confusion matrix over event IDs (harmful = regression, sneak-insert child,
new_deviation, unacceptable counter; benign = concession, acceptable counter,
canary change, sneak-accept child):

- safety_recall = harmful correctly rejected/flagged ÷ all harmful
- restraint = benign correctly accepted ÷ all benign
- rejection_precision = correct rejections ÷ all reject dispositions (0 if none)
- counter_judgment = correct counter decisions ÷ all counters
- continuity = correct prior_position_id links ÷ dispositions requiring one
- grounding = ONLY for correctly-disposed events, quote must equal the
  designated after-span at the canonical pointer (view, change_id, side,
  offsets) — before-text and phase-1 views never ground a phase-2 rejection
- redline_quality = M2 tiered match of agent-authored proposed_redline vs
  signed alternatives, on correct rejections only (the matcher's sound
  direction)
- conformance / abstention as today, with missing-info items carrying opaque
  IDs and abstention scored as precision/recall over those IDs [F5]

Composite = (weighted sum of the above) × harmonic_mean(safety_recall,
restraint), zero if either is zero. The joint gate is the anti-blanket
mechanism: reject-all zeroes restraint, accept-all zeroes safety, and no
other channel can buy either back. Weights proposed at red-pen time, after
pilot mechanical numbers exist.

`all_concessions` is its own task_type [F14]: every change_id must carry an
explicit accept, engagement = all change blocks read AND all change_ids
decided (event coverage, not doc-touch), interactive/card equality required;
V13's zero-deviation clean invariant is untouched.

## Coverage matrix, not sampled weights [F18]

Every mixed task contains ≥1 concession, ≥1 regression, ≥1 sneak, ≥1
new_deviation, one acceptable AND one unacceptable counter, and one
benign_canary_change per tranche minimum. Held-out: ≥2 per practice area,
grouped by base scenario, averaged within before across. No silently absent
channel — fixed quotas or a predeclared N/A rule locked before evaluation.

## Legitimacy battery (pre-spend, deterministic, all must fail) [F17]

accept_all · reject_all_nearest_rule · prior_location_replayer ·
always_reject_counter · turn2_only · duplicate_action_washer ·
before_text_quoter. Each <0.40 composite AND ≤60% of engaged honest; per-event
confusion matrices published so a composite cannot hide which shortcut
worked. Only after these: honest arms (spend-gated).

## Attorney inputs (the only new human work)

Per eligible rule (2–4 per playbook): counter template FAMILIES
(acceptable + unacceptable, slot-renderable) signed at family level, plus
optional context guards (e.g. launch_urgency=high activates a fallback but
never relaxes canaries) [F19]. Everything else derives.

## Build order

1. Schemas + validators (ledger, V14, action/card, V9 extension) —
2. generator response-patch engine (typed ops; sneak = source effect +
   destination templates + grammar slots) —
3. env two-phase + review_change —
4. scoring —
5. mechanical battery —
6. instances (coverage matrix) + model-review QA + rulings —
7. honest arms (spend gate).

## Non-goals (unchanged, now stated honestly) [F20]

No .docx, no live counterparty, no judge in reward. This tier measures
deterministic patch disposition and cross-turn consistency — not Word-edit
surgicalness or open-ended deal-closing prose. T3 (multi-document) remains a
separate, unclaimed tier.
